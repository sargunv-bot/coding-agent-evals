# Calibration evidence

Files under `calibration/` are generated by `scripts/calibrate.sh`.
They contain verifier-control runs, not coding-agent benchmark scores.

For each task or scenario, calibration requires:

- a no-op tree to fail with a valid reward receipt;
- the evaluator-only gold patch to pass twice;
- every plausible-wrong mutant to apply cleanly and fail;
- no infrastructure-error classification.

`calibration/summary.json` is produced only after every expectation passes. Raw records retain verifier runtime and the ignored local `.runs/` receipt path for forensic lookup on the machine that generated them.

Candidate model runs are intentionally absent from this directory.
